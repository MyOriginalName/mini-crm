# Правила кодирования Mini-CRM

## Общие принципы
- SOLID
- DRY (Don't Repeat Yourself)
- KISS (Keep It Simple, Stupid)
- PSR-12 для PHP кода
- ESLint для JavaScript/TypeScript

## Структура кода
### Backend (Laravel)
- Использовать сервисный слой
- Следовать REST API конвенциям
- Использовать ресурсы для API ответов
- Валидация через Form Requests

### Frontend (React)
- Компонентный подход
- Хуки для управления состоянием
- Модульная структура

## Безопасность
- Валидация всех входных данных
- Защита от SQL-инъекций
- CSRF защита
- XSS защита
- Правильное хранение паролей

## Тестирование
- Unit тесты для критической логики
- Feature тесты для API
- Тестовое покрытие > 80%

## Документация
- PHPDoc для PHP классов
- JSDoc для TypeScript/JavaScript
- README для каждого модуля
- Комментарии для сложной логики

## Правила работы с пользователями

1. Каждый пользователь должен иметь как минимум одну роль
2. Роли пользователей не могут быть изменены самостоятельно
3. Только администратор может управлять ролями пользователей
4. Пользователи могут видеть свои роли в профиле
5. При создании нового пользователя необходимо назначить ему роль
6. Удаление роли пользователя возможно только администратором
7. Изменение ролей пользователя должно логироваться
